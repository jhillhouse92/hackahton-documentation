Architecture Overview

The VA Application Architecture involves 3 components:

  1. The Application - HTML/JS web application (front-end code)
  2. The Middle Tier - Note: If using MDWS (pronounced Meadows), a SOAP client will be needed to interface between the Application and MDWS.
  3. The Database
    • VistA - A database of one hospital facility
    • CDW - A collection of all VistA instances across multiple hospital facilities
    • MongoDB - Application database used for individual application needs
